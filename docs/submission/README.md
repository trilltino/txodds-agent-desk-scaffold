# docs/submission

Submission docs are written for demos, judges, and product review rather than day-to-day implementation.

## Files

- `demo-script.md`: walkthrough script for showing the app.
- `judging-map.md`: how the app maps onto judging criteria.
- `product-spec.md`: concise product framing.

## Rules

- Keep claims aligned with what the code actually does.
- Mark scaffolded or mock behavior clearly when a live integration is not enabled.
